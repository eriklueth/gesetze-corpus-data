# gesetze-corpus-data

**Versioned Markdown corpus of German federal law** — one commit per effective date, backdated to the `stand_datum` reported by [gesetze-im-internet.de](https://www.gesetze-im-internet.de). `git log` on any paragraph file is therefore a first-class history of that section:

```bash
git log --follow --oneline -- laws/BJNR010050934/paragraphs/0007g.md
```

This repo contains **only data**, never code. The pipeline that produces it lives in [`gesetze-online-corpus`](https://github.com/eriklueth/gesetze-online-corpus).

## Layout

```
laws/<BJNR>/
    meta.json                 # law-level metadata (jurabk, title, stand_datum, sha256, …)
    toc.json                  # canonical ordered section list with file pointers
    source.xml                # canonicalized GII XML (c14n2)
    paragraphs/<NNNN[x]>.md   # one file per § or Artikel
    annexes/<NNNN[x]>.md      # one file per Anlage

events/<YYYY>/<event_id>.json # one file per grouped effective-date change

sources/current/gii-index.json
    # sha256 per law slug, used to detect upstream changes between snapshots

derived/                      # regenerated by `gesetze-corpus export`
    by-jurabk.json            # {jurabk: bjnr}
    by-bjnr.json              # {bjnr: {jurabk, title, stand_datum, section_counts}}
    corpus.jsonl              # one JSON line per rendered section
```

Full schema: [`SCHEMA.md`](SCHEMA.md).

## What a section looks like

```markdown
---
schema_version: v2
bjnr: BJNR001950896
jurabk: BGB
type: paragraph
number: "§ 14a"
heading: Begriffsbestimmung
breadcrumb:
  - Buch 1
  - Abschnitt 3
stand_datum: "2024-01-01"
source_xml: source.xml
---

# § 14a Begriffsbestimmung

(1) Erster Absatz.

(2) Zweiter Absatz.
```

Every paragraph file is self-describing: the YAML front-matter carries enough metadata to identify the law, section, breadcrumb, and the effective date of the version captured in that commit.

## How to use the corpus

### Clone it directly

```bash
git clone https://github.com/eriklueth/gesetze-corpus-data.git
```

At ~200 MB the clone is modest; a shallow clone (`--depth 1`) drops history and lands in well under a minute.

### Grep the full corpus

```bash
jq -r 'select(.text | test("Investitionsabzug")) | [.jurabk, .number] | @tsv' \
    derived/corpus.jsonl
```

### Look up a law by its jurabk

```bash
jq -r '."EStG"' derived/by-jurabk.json
# → BJNR010050934
```

### See which laws changed on a given day

```bash
git log --since=2026-04-01 --until=2026-04-10 --oneline
```

### Read a section at a past effective date

```bash
git show <commit>:laws/BJNR001950896/paragraphs/0014a.md
```

## Automation

This repo is **fully automated**:

- Every commit is written by the pipeline (`law(...): stand <date>` for content, `chore(sync): …` for bookkeeping).
- History is **never rebased** after the initial snapshot — commit SHAs are permanent.
- Pull requests on the data itself are not accepted; report data issues against [`gesetze-online-corpus`](https://github.com/eriklueth/gesetze-online-corpus/issues) instead.
